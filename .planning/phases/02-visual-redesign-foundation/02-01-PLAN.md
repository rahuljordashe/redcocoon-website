<task type="auto">
  <name>Task 3: Audit all component style blocks for hardcoded color values (VIS-02)</name>
  <files>none modified — audit only, results documented in SUMMARY</files>
  <action>
Run a grep-based audit of ALL component `<style>` blocks for hardcoded color values (hex codes AND rgba values matching palette colors). This ensures VIS-02 coverage extends beyond the components modified in Plan 02.

**Step 1: Run the audit command:**
```bash
grep -rn '#[0-9A-Fa-f]\{3,6\}\b\|rgba\?' src/components/*.astro | grep -v 'node_modules'
```

**Step 2: Classify every match into one of these categories:**

**ACCEPTABLE — Leave as-is (document in SUMMARY):**
- Third-party brand colors: WhatsApp green `#25D366` / `#1da851`, Instagram purple `rgba(131, 58, 180, ...)` — these are NOT our palette
- White text on colored backgrounds: `#fff` / `color: #fff` on terracotta/WhatsApp/Instagram buttons — standard practice
- Opacity/transparency overlays using rgba of palette colors: e.g., `rgba(42, 36, 32, 0.75)` (espresso at 75% for overlay), `rgba(253, 251, 247, 0.92)` (off-white at 92% for navbar glass), `rgba(212, 200, 181, 0.3)` (beige-dark at 30% for border), `rgba(255, 255, 255, 0.08)` (subtle white overlay on dark bg) — these require alpha transparency and cannot use simple CSS tokens
- Generic black shadows: `rgba(0, 0, 0, 0.xx)` — standard shadow practice, already being tokenized where appropriate in Plan 02

**NEEDS TOKEN — Flag for Plan 02 to fix:**
- Any bare palette hex used as a flat color (not in rgba for opacity): e.g., `color: #2A2420` instead of `var(--color-espresso)`, `background: #F5F0E8` instead of `var(--color-cream)`
- Any `background: white` or `color: white` that should use `var(--color-white)` (Plan 02 Task 3 already fixes ProductCard's `background: white`)

**Step 3: Specifically check these components NOT in Plan 02's file list:**

- `FloatingContactButton.astro` — expected: `rgba(131, 58, 180, ...)` Instagram purple shadows, `#fff` text, `rgba(0, 0, 0, 0.15)` shadow (ACCEPTABLE — brand colors + standard shadows)
- `MobileMenu.astro` — expected: no hardcoded colors (CLEAN)
- `BottomTabBar.astro` — expected: no hardcoded colors (CLEAN)
- `BackToTop.astro` — expected: `rgba(193, 119, 71, 0.3)` terracotta shadow (ACCEPTABLE — opacity overlay)
- `Footer.astro` — expected: `rgba(255, 255, 255, 0.08)` and `rgba(107, 91, 78, 0.3)` (ACCEPTABLE — opacity overlays on dark bg)

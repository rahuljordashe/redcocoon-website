**Step 2: Update Hero.astro**

Add import in frontmatter:
```astro
---
import { Image } from 'astro:assets';
import heroImg from '../assets/images/399x650-01.png';
---
```

Replace the `<img>` tag:
```astro
<Image
  src={heroImg}
  alt="Handcrafted ceramic leaf plates"
  class="hero-img"
  width={399}
  height={650}
  loading="eager"
/>
```

Note: Hero image uses `loading="eager"` (not lazy) because it is above the fold and already preloaded. Also update the `<link rel="preload">` in BaseLayout.astro — since the image is now processed by Astro (WebP output with hashed filename), the preload href will no longer match. REMOVE the preload link from BaseLayout.astro (`<link rel="preload" as="image" href="/images/399x650-01.png">`) since Astro Image handles this via width/height attributes (no CLS) and the image is inlined in the initial HTML with `loading="eager"`.

**Step 5: Update Process.astro**

This component uses a `map()` with dynamic image paths. Since Astro's `<Image />` requires static imports (not dynamic strings), use a different approach. Import all 4 process images statically, create a lookup map:

```astro
---
import { Image } from 'astro:assets';
import SectionHeader from './SectionHeader.astro';
import platesImg from '../assets/images/Plates1.jpg';
import coastersImg from '../assets/images/Coasters18.jpg';
import toastPlatesImg from '../assets/images/Toast-Plates-new.jpg';
import fireGlowImg from '../assets/images/Fire-Glow.jpg';

const imageMap: Record<string, ImageMetadata> = {
  'Plates1.jpg': platesImg,
  'Coasters18.jpg': coastersImg,
  'Toast-Plates-new.jpg': toastPlatesImg,
  'Fire-Glow.jpg': fireGlowImg,
};

const steps = [
  { number: '01', name: 'Shaping', image: 'Plates1.jpg', desc: '...' },
  // ... same as current
];
---
```

In the template, replace `<img>` with:
```astro
<Image src={imageMap[step.image]} alt={step.name} width={300} height={225} loading="lazy" class="step-img" />
```

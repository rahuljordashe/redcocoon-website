**imageMap:** Same Record<string, ImageMetadata> pattern as GalleryGrid, mapping '/images/Bowls5.jpg' -> imported module, etc.

**getStaticPaths:**
```typescript
export async function getStaticPaths() {
  const products = await getCollection('products');
  return products.map(product => ({
    params: { slug: product.id },
    props: { product },
  }));
}
```

**Page logic:**
```typescript
const { product } = Astro.props;
const { name, code, price, currency, collection, material, stock, images, madeToOrder } = product.data;
const isSold = stock === 0;
const isMTO = madeToOrder === true;

// Related products
const allProducts = await getCollection('products');
const related = allProducts
  .filter(p => p.data.collection === collection && p.id !== product.id)
  .slice(0, 3);

// Image
const mainImage = imageMap[images[0]];
const dmLink = getInstagramDMLink();
const reserveLink = getReserveLink(name, price, currency);
```

```html
    <li aria-current="page">{name}</li>
  </ol>
</nav>
```

2. **Product detail section** — side-by-side layout:
```html
<div class="product-detail">
  <div class="product-image">
    <a href={images[0]} class="glightbox" data-gallery="product">
      <Image src={mainImage} alt={name} width={600} height={750} loading="eager" />
    </a>
  </div>
  <div class="product-info">
    <span class="product-code">{code}</span>
    <h1 class="product-name">{name}</h1>
    <span class="product-price">{formatPrice(price, currency)}</span>

      <div class="product-actions">
        <a href={dmLink} target="_blank" rel="noopener noreferrer" class="btn btn-instagram">DM to Reserve</a>
        <a href={reserveLink} target="_blank" rel="noopener noreferrer" class="btn btn-whatsapp">WhatsApp Us</a>
      </div>
    )}
  </div>
</div>
```

Note: Related products use ProductCard which still uses `<img>` with string paths. This is acceptable — the main product image uses Astro Image (GAL-07); related cards are secondary and will be addressed in Phase 5 optimization if needed.

4. **GLightbox script:**
```html
<script>
  import GLightbox from 'glightbox';
  document.addEventListener('DOMContentLoaded', () => {
    GLightbox({
      selector: '.glightbox',
      touchNavigation: true,
      loop: false,
      autoplayVideos: false,
    });
  });
</script>
```

**IMPORTANT for GLightbox href:** The `<a href>` on the lightbox trigger must point to a URL that resolves to an image. Since we're using Astro Image (which outputs to `/_astro/*.webp`), the `href` should use the public/ path (`/images/Bowls5.jpg`) which still exists for direct access. GLightbox will load that URL in the overlay. Alternatively, if the Astro Image output URL is needed, you can extract it from the Image component's output — but using the public/ path is simpler and works.

**Vanilla JS `<script>` block for filtering:**
Implement the filter logic per RESEARCH.md Pattern 3:
1. Query all `.filter-pill` buttons, `.gallery-item` elements, and `.gallery-count` element
2. On pill click:
   - Remove `.active` from all pills, add to clicked pill
   - Get `data-collection` value from clicked pill
   - For each `.gallery-item`:
     - If collection is "all" or matches item's `data-collection`: show (opacity 1, display block after a brief timeout)
     - Else: hide with fade (opacity 0, then display none after 300ms via transitionend listener)
   - Update count text: empty for "all", "Showing N piece(s)" for filtered view
3. Use the two-step fade approach from RESEARCH.md to avoid display:none killing transitions:
   - To hide: set `opacity: 0`, then on `transitionend` set `display: none`
   - To show: set `display: ''` (block), then requestAnimationFrame -> set `opacity: 1`
   - Guard against rapid clicking: track current filter, cancel pending transitions

**Scoped styles:**
- `.shop-gallery`: `padding: var(--space-5xl) 0; background: var(--color-white);`
- Page-level responsive adjustments if needed
